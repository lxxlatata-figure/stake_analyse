# 项目技术框架与主要难点说明

## 1. 项目目标

该项目用于自动抓取杭州上市公司财报 PDF，提取关键财务信息，调用 DeepSeek 生成偏向股价预估的分析结果，并通过前端页面展示 PDF 列表与分析结论。

当前版本重点解决了四个问题：

1. 把原先需要分别运行的抓取脚本和分析脚本整合为统一流程。
2. 把页面从后端拼接 HTML 改为 `Flask API + Vue 前端` 的结构。
3. 把 DeepSeek 输入从“整份 PDF 开头少量文本”改为“关键页提取 + 本地摘要 + 结构化提示词”。
4. 通过并发抓取、快速文本提取、缓存跳过重复分析，提升整体效率。

## 2. 技术框架

### 后端

- 语言：Python 3.9
- Web 框架：Flask
- 网络请求：requests
- PDF 文本提取：PyPDF2
- 并发模型：`concurrent.futures.ThreadPoolExecutor`
- 数据持久化：本地 JSON 文件 `data/report_results.json`

### 前端

- 框架：Vue 3
- 组织方式：单页界面，通过 Flask 模板加载 Vue 页面
- 样式方案：纯 CSS，自定义黑白灰视觉风格
- 数据交互：前端轮询调用 Flask API

### 外部数据与模型

- 财报来源：巨潮资讯网公告接口
- 大模型：DeepSeek Chat API

## 4. 核心流程

### 4.1 财报抓取流程

1. 读取 `hangzhou_companies.txt` 中的股票代码。
2. 对每个股票代码并发请求巨潮接口。
3. 使用“年度报告 / 半年度报告 / 第一季度报告 / 第三季度报告”作为关键词检索。
4. 过滤掉摘要、英文版、提示性公告、制度文件、补充公告等非正式财报正文。
5. 下载 PDF 到 `pdfs/` 目录。
6. 下载成功后立即写入结果文件，前端即可显示。

### 4.2 文本提取与摘要流程

1. 使用 `PyPDF2` 读取 PDF。
2. 不再全文提取，而是优先提取前几页、关键业务页和结尾风险页。
3. 从提取文本中抓取包含收入、净利润、现金流、ROE、研发投入等关键词的行。
4. 组合成“核心财务指标 + 经营与风险要点 + 关键片段”的本地摘要。

### 4.3 DeepSeek 分析流程

1. 将公司名称、财报标题、本地摘要、关键片段一并送入 DeepSeek。
2. 输出固定 JSON 结构：
   - `price_view`
   - `expected_range`
   - `confidence`
   - `short_reason`
   - `bull_points`
   - `bear_points`
   - `price_prediction`
3. 分析结果写回本地 JSON，前端折叠卡片直接读取展示。

### 4.4 前端展示流程

1. Vue 页面轮询 `/api/state` 获取任务状态。
2. Vue 页面轮询 `/api/reports` 获取当前记录列表。
3. 左侧列表展示已抓取 PDF。
4. 右侧只展示 `stage=done` 的分析结果。
5. 分析结果使用 `details/summary` 折叠结构，默认只显示标题、股价倾向、预估区间、置信度。

## 5. 主要难点

### 难点 1：巨潮搜索结果不稳定

直接按股票代码搜索时，返回结果往往混有大量日常公告，真正财报会被淹没。

解决方式：

- 使用“股票代码 + 财报关键词”的组合搜索。
- 对标题做二次过滤，只保留正式财报。

### 难点 2：PDF 很长，全文提取太慢

很多半年度报告和年度报告页数很大，如果全文提取，单份报告就可能耗时很长，导致整体任务卡住。

解决方式：

- 改为快速提取关键页。
- 对单页字符数做截断。
- DeepSeek 输入只保留高价值片段，而不是整份财报全文。

### 难点 3：模型结果不稳定

如果把原始 PDF 大段文本直接交给模型，容易出现：

- 只关注开头目录和声明
- 输出过于泛化
- 返回格式不稳定

解决方式：

- 先在本地提炼指标和片段，再交给模型。
- 强约束输出为 JSON。
- 把提示词聚焦到“未来 1-3 个月股价预估”。

### 难点 4：前端需要实时感知任务进度

抓取和分析是后台异步任务，如果前端只在任务结束后整体刷新，用户体验会很差。

解决方式：

- 拆分状态接口和结果接口。
- 后端每处理完一份 PDF 就写入结果缓存。
- 前端轮询接口，实现“抓一份显示一份，分析完一份显示一份”。

### 难点 5：并发写本地 JSON 容易冲突

抓取线程和分析线程都可能同时更新同一个结果文件。

解决方式：

- 使用线程锁保护读写。
- 采用按股票代码、标题、公告日期进行 `upsert`。

## 6. 当前版本的优点

- 单入口运行，流程统一
- 前后端分层更清晰
- 页面视觉更完整，交互更符合展示场景
- 分析速度较之前明显提升
- 支持选择抓取 5 份、10 份或全部
- 结果默认折叠，适合大量公司同时展示

## 7. 后续可以继续优化的方向

### 方向 1：引入 OCR

如果部分 PDF 为扫描版，`PyPDF2` 无法稳定提取文字，可以补充 OCR 识别流程。

### 方向 2：加入真实市场价格数据

当前“股价预估”是相对当前股价的区间判断，没有直接接股票行情接口。

后续可以接入：

- 东方财富接口
- Tushare
- akshare

这样可以把“预估区间”改成更接近真实市场价格的目标价表达。

### 方向 3：前端工程化

当前 Vue 页面是轻量集成方式，适合课程项目快速落地。

后续如果继续扩展，可以改成：

- `Vue + Vite`
- 独立前端工程
- axios + 状态管理
- 图表库展示趋势和分类结果

### 方向 4：结果数据库化

如果后续分析记录变多，建议从本地 JSON 文件切换到数据库，例如：

- SQLite
- MySQL

这样更适合分页、筛选、搜索、历史对比和多用户访问。
